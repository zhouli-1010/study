Client Side 開発環境の構築¶

インストール¶

HUE - Build Client Environment

追加で以下のプログラムのインストールが必要になります。

ソフト名 推奨バージョン インストール方法 （ダウンロードサイト、npm コマンド） 注意点
Gulp 3.8.10 npm install -g gulp@3.8.10 Console2 上で実行してください。

警告

Console2を起動するには必ず C:\HUE\WorkSpace\Develop\console2.bat を実行してください。 C:\HUE\Application\Console2-<version>\Console.exe を起動すると、 環境の設定が行われないため、コマンドが正しく実行できません。

警告

HUE アカウントが発行されていない場合は、この方法での HUE 環境構築は出来ません。 以下の「HUE アカウントを持っていない場合」に従ってセットアップを行ってください。

HUE アカウントを持っていない場合 (Windows)¶

HUE アカウントを持っていない場合は、以下のプログラムを手動でインストールする必要があります。 ただし、パスの設定などトラブルが多いので、頑張ってください。

ソフト名 推奨バージョン インストール方法 （ダウンロードサイト、npm コマンド） 注意点
Java 8 最新版    
Python 2 2.7.5-2.7.7 https://www.python.org/download/releases/2.7.6/ Python 3 系では動作しません ので注意してください。 Python 2 および Python 3 とが共存する環境で実施する場合、 Python コマンドをフルパスで指定するか、Python コマンドへのパスを適切に切り替えてください。
Node.js v0.10.15-v0.10.33 http://nodejs.jp/nodejs.org_ja/docs/v0.10/#download  
npm 1.2.0-1.4.28 Node.js インストール時に同時にインストール  
Git 1.8.5-1.9.5 Git for Windows <https://msysgit.github.io/> Git for Windows の代わりに Cygwin を使っても動作すると思われます(未検証)。
Gulp 3.8.10 npm install -g gulp@3.8.10 Git をインストールした際に同時にインストールされる Console2 (Cygwin Terminal も可。以下の説明では Console2 に統一) 上で実行してください。

ノート

「Console2」は「Git BASH」もしくは「Cygwin Terminal」と読み替えてください

ノート

これ以外のバージョンでもおそらく動作すると思われます。

HUE アカウントを持っていない場合 (Linux)¶

Linux の場合は、追加でインストールするソフトウェアは少なくなります。

ソフト名 推奨バージョン インストール方法 （ダウンロードサイト、npm コマンド） 注意点
Java 8 最新版   Oracle JDK を推奨。
Node.js v0.10.15-v0.10.33 sudo apt-get install nodejs (deb系) sudo yum install nodejs (rpm系)  
npm 1.2.0-1.4.28 sudo apt-get install npm (deb系) sudo yum install npm (rpm系)  
Git 1.8.5-1.9.5 sudo apt-get install git gitk (deb系) sudo yum install git (rpm系)  
Gulp 3.8.10 npm install -g gulp@3.8.10 npm をインストールした後に端末より実行します。

ノート

「Console2」は利用中の端末プログラム名に読み替えてください。

ノート

apt-get install もしくは、yum install で該当パッケージが存在しないエラーがでるかもしれません。 適宜ダウンロード元のリポジトリを追加してください。

課題の展開¶

演習及び判定試験で配布している zip ファイルを適当なディレクトリに展開します。

ノート

ここでは、C:\Users\works\hue-bootcamp\ ディレクトリ以下に展開されるものとします。 必要に応じて適宜読み替えてください。

警告

Eclipseのワークスペースディレクトリには展開しないでください。 いったん別のディレクトリに展開しておいて、後でEclipseにインポートします。

初期化¶

  1. Console2 より、展開した zip ファイルの課題ディレクトリ (例: cs-i-practice) に移動する

警告

Console2 は console2.bat から起動してください。 そうしないと環境変数がセットされず色々なものが動作しません。
$ cd /c/Users/works/hue-bootcamp/cs-i-practice/

ノート

package.json および gulpfile.js が保存されているディレクトリです

ノート

「$」はプロンプト文字列 (入力受付中を意味する、BASH が出力する文字) は入力しません。

  1. 以下の初期化コマンドを実行する
$ npm install
$ gulp init

ノート

Closure Libraryの取得などが行われます。

ノート

npm install 実行時に、次のような警告が表示されます。 この警告は無視して問題ありません。

Eclipse プロジェクトの作成¶

Eclipse で、新しい JavaScript プロジェクトを作成します。 以下の手順で作成してください。

  1. Eclipseで新しいJavaScript Projectを作成
    • [Project name]以外の設定は変更せずに[Finish]ボタンを押下し、 空のJavaScriptプロジェクトを作成してください。
  2. [Project Explorer]でプロジェクトを右クリックして[Properties]を選択
  3. [JavaScript]＞[Include Path]を選択し、[Source]タブを選択
  4. [Source files and folders in the Global Scope:]で[Excluded: (None)]を選択し、[Edit]ボタンを押下
  5. 二つある[Add]ボタンのうち、下の方の[Add]ボタンを押下
  6. [Exclusion patter]に「node_modules/**/*.js」と入力して[OK]ボタンを押下
  7. [Finish]ボタンや[OK]ボタンを押下してすべてのダイアログを閉じる
  8. [Project Explorer]でプロジェクトを右クリックして[Import]を選択
  9. [General]>[File System]を選択して[Next]ボタンを押下
  10. [From directory]に C:\Users\works\hue-bootcamp\cs-i-practice を指定
  11. cs-i-practice のディレクトリツリーが表示されるので[cs-i-practice]ディレクトリをチェックして[Finish]ボタンを押下

ノート

Eclipse 以外の任意のテキストエディタを使用しても構いません。

Auto formatter setting XML for HUE JavaScript¶

HUE JavaScript Formatter Setting

  1. Download Setting file (Named HUE_JavaScript.xml)
  2. Start your Eclipse
  3. Window > Preference > JavaScript > Code Style > Formatter
  4. Import setting file
  5. Select profile COMPANY js format
  6. Try Ctrl+A -> Shift+Ctrl+F to format your JavaScript file

ノート

In your environment, this file may be imporeted by Environment Manager. If you don’t have this setting, please try this steps.

Eclipse GJSLintプラグインのインストールと設定¶

プラグインのインストールをします。

  1. GJSLintプラグイン をダウンロード
  2. zipファイルを適当なフォルダに展開
  3. com.worksap.bootcamp.eclipse_<version>.jarをEclipseのpluginsディレクトリ(Ex: C:\Develop\Application\eclipse-4.4.0-20141017\eclipse\plugins )にコピー
  4. Eclipseを起動
  5. メインメニューの[Window]>[Preferences]を選択
  6. [WAP BC GJSLint]を選択
  7. [gjslint path]にgjslint.exeのパス(Ex: C:\Develop\Application\python-27\Scripts\gjslint.exe )を入力

インポートしたプロジェクトの設定をします。

  1. [Package Excplorer]でプロジェクトを右クリック
  2. [Configure]>[Enable WAP BC GJSLint]を選択
  3. [Package Excplorer]でプロジェクトを右クリック
  4. [Properties]を選択
  5. [WAP BC GJSLint]を選択
  6. [Inclusion patterns]に”**.js”、[Exclusion patterns]に”**/node_modules/**, **/lib/**, **/webeditor.js, **/todolist.js, **/gulpfile.js”入力 (最初と最後の「”」は入力しないでください)

課題のビルド¶

Google Closure Compilerによるコンパイルは、Console2 の課題ディレクトリ上で次のコマンドを実行します。

$ gulp build

ノート

加えて、ソースコードマップの生成 (ブラウザのデバッガでコンパイル前のソースコードを用いてデバッグする機能) など、 より快適に課題を進めることができるようになっていますので、積極的に活用してください。

課題のチェック¶

課題のチェックは、Console2 の課題ディレクトリ上で次のコマンドを実行します。

$ gulp check

ノート

判定試験課題は、このチェックコマンドにより一切の警告およびエラーが発生しないことが最低条件になります。 何らかの警告やエラーが発生した場合、機械的に不合格になり、以降の評価が行われないことに注意してください。

なお、このチェックにはビルドチェック (gulp build コマンド) も含まれています。

課題の提出¶

判定試験課題は、Console2 の課題ディレクトリ上で次のコマンドを実行します。

$ gulp package

同じディレクトリに生成された zip ファイルを、研修管理システムなどより提出してください。

ノート

この時、同時にエラーチェックが行われます。必ずチェックが通っていることを確認してください。

トラブルが発生したら¶

calcdeps.py: Finding Closure dependencies... の後に Traceback (most recent call last): から始まるエラーメッセージが表示される

「最後にインストールした」Java が古いバージョンの場合 （例えば、Java 8 をインストールした後、プロダクトの開発環境をセットアップした場合など）、 Closure Tools の python コマンドから java コマンドの実行に失敗するケースがあります。 (エラーメッセージに proc = subprocess.Popen(['java', '-version'], stderr=subprocess.PIPE) の行が見つかるはずです)

この場合、Java 8 を一度アンインストールしてから、再度 Java 8 をインストールし直してください。

ノート

Java は基本的には後方互換性を持っているため、Java 8 で既存プロダクトが動作しないとは考えにくいですが、 どうしても Java 8 のインストールが出来ない場合は相談してください。

'python' □□A□□... のような文字化けしたエラーメッセージが表示される

Windows の環境変数 PATH に python のプログラムが見つからない可能性が高いです。 環境変数を設定して「コマンドプロンプト」から python --version コマンドが正常実行できるようにしてから 再度実行してください。

ソースコードに何らかのトラブルが発生した場合、Console2 の課題ディレクトリ上で次のコマンドを実行します。

$ gulp clean

ノート

gulp build コマンドで生成されたファイルが削除されます。

これでも解決しない場合は、以下のコマンドを実行します。

$ gulp clean:all
$ gulp init

ノート

gulp clean コマンドに加え、gulp init コマンドで取得・生成した初期ファイルが削除されます。 このため、再び gulp init コマンドの実行が必要になります。

ノート

いずれのコマンドも、基本的にはみなさんが変更を加えたファイルをリセットすることはありませんが、 念の為に、コマンド実行前にはバックアップ (課題ディレクトリのコピー) を取っておくことをおすすめします。 Git (など) でのソースコードのバージョン管理の練習を行うのも良いでしょう。
