UI コンポーネントを使ってみよう¶

Closure Library では、DOM 操作を行うプログラムを UI コンポーネントとして分離しています。

Closure Library には、大量の UI コンポーネントが用意されています。 下記のサイトにアクセスしてみて、実際にどのようなコンポーネントがあるのかを見てみましょう。

http://closure-library.googlecode.com/git-history/0148f7ecaa1be5b645fabe7338b9579ed2f951c8/closure/goog/demos/index.html

HUE 開発においては、wap.core.ui 名前空間以下に様々な UI コンポーネントが用意されています。

ノート

HUE Bootcamp では、HUE Framework の UI コンポーネントは利用しません。 Closure Library の提供するコンポーネントもしくは、皆さんが自作したコンポーネントを利用します。

UI コンポーネントことはじめ¶

Todo List でいきなり UI コンポーネントを使う前に、安全な場所で練習します。 練習用の場所を、sandbox ディレクトリに用意しています。

まずは sandbox/html/hello-ui.html をエディタ (Eclipse) とブラウザ (Google Chrome) で開いてみましょう。

将来ボタンになりそうな何かと、将来選択リストになりそうな何かが並んでいます。 現時点ではもちろん動作しませんが、これを動かすのがゴールです。

ノート

sandbox では、すぐ書いてすぐ動かすを実現するために、 JavaScript ソースコードを HTML の <script> 要素内に直接記述するようにしていますが、 実際の課題および判定試験、そしてもちろん、本番の HUE 開発では <script> 要素内に記述するのは厳禁ですよ。

ボタンを動かしてみよう¶

Closure Library には、ボタンに対応するコンポーネントとして、goog.ui.Button が用意されています。 これを使ってみます。

hello-ui.html の <script> 内に、次のコードを書いてみます。

var button = new goog.ui.Button();
var $button = goog.dom.getElement('hello-button');
button.decorate($button);

これが基本的な使い方になります。

ノート

なお、この書き方は最新の Closure Library では非推奨とされています。 代わりに、次のように記述することになっています。

var $button = goog.dom.getElement('hello-button');
var button = goog.ui.decorate($button);

これは、コンポーネントのインスタンス選択を goog.ui.decorate() (内部で利用している goog.ui.registry ) に任せることで、 コンストラクタ呼び出しコードと HTML のクラス名とのダブルメンテナンスを防ぐことを目的としていると考えられます。

しかし、コンポーネントのインスタンス選択の実装が現時点ではループ処理 (比較的低速) によって行われていること、 プラグインを取捨選択して decorate() することに対応していないこと、 HUE Bootcamp で作成するコンポーネントは decorate() が非推奨とされている goog.ui.Control クラスではなく goog.ui.Component クラスを継承して作成している関係上 (つまり、HUE Bootcamp のコンポーネントに限って言えば非推奨ではない)、 HUE Bootcamp では @deprecated な古い書き方を利用します。

HUE Framework の対応次第では、HUE 開発における推奨方法が変わる可能性があることに注意してください。

ノート

decorate() の他に UI コンポーネントを生成するもうひとつのやり方に、render() があります。 render() は、decorate() すべき HTML 要素がない状態のところに、新しい HTML 要素を生成してからdecorate() します。

一般に DOM 操作は慎重に行う限りは「それほど」重たい操作ではありませんが、 HUE においてはそれすら命取りとなりかねません。

特に、HUE 開発においては、HTML の組み立てを出来る限りサーバーサイドで行うという思想で設計されているので、 render() を忘れてしまっても良いでしょう。 HUE の場合の原則として、「ページのロード時には一切の DOM の変更は行わない」ということを覚えておいてください。 ただし、HUE Bootcamp では、HUE Framework は利用していないので、サーバーサイドの支援を受けることが出来ません。 そのため、ページロード時の DOM 変更を一部だけ許容していることには注意してください。

ノート

Closure Library では、decorate() メソッドは HTML 要素を修飾 (decorate) することを前提としています。 幾つかの UI component では decorate() 時に renderer を引数に取って見た目を変える設計になっていますが、 HUE ではこのような設計を行わない (ページロード時には DOM の変更を行わない) ので注意してください。

この状態でブラウザを更新してみても、まだ何も起こりません。

次に、ボタンをクリックしたら、メッセージを表示するようにします。

イベントが発生した時に処理を行うためには、goog.events.listen を使い、 イベントハンドラをブラウザに登録する必要があります。 先ほどのコードの後ろに次のコードを追加してみましょう。

function showMessage(event) {
  alert('Hello, UI component!');
}
goog.events.listen(button, goog.ui.Component.EventType.ACTION, showMessage);

ページを更新し、ボタンをクリックしてみましょう。 画面に「Hello, UI component!」というメッセージが表示されましたね。

goog.events.listen の引数は次の通りです。

  • 第1引数: イベントの対象となるコンポーネント。ここでは button を指定しています。

  • 第2引数: イベントの種類。ボタンをクリックした際には、goog.ui.Component.EventType.ACTION イベントが発生します。

  • 第3引数: イベント発生時の処理 (コールバック関数)。ここでは showMessage 関数が指定されています。

    無名関数で直接書き下すことも出来ますが、極力名前を付けるようにしてください。 イベント用のコールバック関数をイベントハンドラとも呼びます。

第3引数に渡すイベントハンドラには、第1引数にイベントの詳細情報が渡されます。 実際にイベント詳細情報を利用するか否かにかかわらず、必ずイベントハンドラには event 引数を定義するようにしてください。 例の showMessage では関数名の直後に (event) と書かれています。このようにするだけで event 引数を定義できます。

警告

Google Chrome では、イベント情報が格納される event グローバル変数 (window.event) が用意されます。 そのため、イベントハンドラに event 変数を定義しなくても、イベント情報にアクセスできるように見えます。 しかしこれは Google Chrome の独自仕様であり、他のブラウザ (例えば Firefox) では利用できません。 このため、全てのイベントハンドラには利用の有無にかかわらず保険的に event 引数を定義するようにします。

選択リストを使ってみよう¶

HUE Bootcamp では、選択リストの UI コンポーネントが用意されています。これを使ってみましょう。

次のコードを追加します。

var list = new wap.bootcamp.common.ui.List([
  wap.bootcamp.common.ui.SelectableListPlugin.getInstance()
]);
var $list = goog.dom.getElement('list');
list.decorate($list);

wap.bootcamp.common.ui.List コンポーネントは、プラグインを追加して機能拡張ができるように作られています。 今回は、選択リストのプラグインを組み込みます。

このコードを追加してからブラウザの Item 1 〜 Item 3 をクリックすると、 選択が切り替わることが確認できます。

イベントをキャンセルする¶

選択リストのリストが切り替わる際には、 wap.bootcamp.common.ui.SelectableListPlugin.EventType.ITEM_SELECT イベントが発生します。 goog.ui.Button と同様に、イベントハンドラを設定してみましょう。

function onItemSelect(event) {
  /** @type {wap.bootcamp.common.ui.ListItem} */
  var listItem = event.target;
  alert('Item "' + listItem.getCaption() + '" is selected.');
}
goog.events.listen(list, wap.bootcamp.common.ui.SelectableListPlugin.EventType.ITEM_SELECT, onItemSelect);

このプログラムを実行して、Item 1 〜 Item 3 をクリックすると、 例えば「Item “Item 1” is selected.」というメッセージが表示されます。

ここでポイントとなるのは、画面にメッセージが表示されているタイミングでは、 まだ UI 上は切り替わっていないことにあります。 つまり、この時点で何らかの事情でイベントをキャンセルすることも出来るのです。

多くのコンポーネントにおいて、標準のイベント挙動をキャンセルする場合は、 event.preventDefault() を呼び出します。

例えば、次のコードを書き足してみてください。 キャプションが「Item 3」のアイテムへの選択をキャンセルします。 その結果、「Item 3」を選択できないようになります。

function onItemSelect(event) {
  /** @type {wap.bootcamp.common.ui.ListItem} */
  var listItem = event.target;

  /** Add from here */
  if (listItem.getCaption() === 'Item 3') {
    event.preventDefault();
    return;
  }
  /** Add to here */

  alert('Item "' + listItem.getCaption() + '" is selected.');
}
goog.events.listen(list, wap.bootcamp.common.ui.SelectableListPlugin.EventType.ITEM_SELECT, onItemSelect);
