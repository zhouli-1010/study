java.nio¶

Reading Text Files / テキストファイルを読み込みたい¶

java.nio package now has new APIs to handle files easily.

java.nioパッケージでは、ファイル取り扱いを容易にするAPIが追加されています。

Old Style¶

In Java 6, you had to write such source code as following to read text files into List<String>.

Java 6では、テキストファイルをList<String>に読み込むために次のようなソースコードを書きました。

/** Old style (Java 6) **/
public static List<String> readFromFile(String fileName) throws IOException {
        FileReader fr = null;
        BufferedReader br = null;
        Scanner scanner = null;
        try {
                fr = new FileReader(fileName);
                br = new BufferedReader(fr);
                scanner = new Scanner(br);

                List<String> result = new ArrayList<String>();
                while (scanner.hasNextLine()) {
                        result.add(scanner.nextLine());
                }
                return result;
        } finally {
                closeSilently(scanner);
                closeSilently(br);
                closeSilently(fr);
        }
}

private void closeSilently(Closeable target) {
        try {
                target.close();
        } catch (IOException e) {}
}

注釈

br.close() and fr.close() are reactively called when you call scanner.close() . Thus, it does not fail to release the resources even without writing closeSilently(br) and/or closeSilently(fr) . However, it is still recommended to write them explicitly.

なお、scanner.close() 時に br.close()、fr.close() が連鎖的に呼び出されるので、 closeSilently(br)、closeSilently(fr) は記述しなくてもリソースの開放漏れが発生することはありませんが、明示的に記述することが望ましいです。

You can write the equivalent code as following, using try-with-resources, which is introduced in Java 7.

Java 7で導入されたtry-with-resourcesを使うと以下のように書くことも出来ます。

/** Old style (Java 7 without java.nio package) **/
public List<String> readFromFile(String fileName) throws IOException {
        try (
                FileReader fr = new FileReader(fileName);
                BufferedReader br = new BufferedReader(fr);
                Scanner scanner = new Scanner(br)
        ) {
                List<String> result = new ArrayList<String>();
                while (scanner.hasNextLine()) {
                        result.add(scanner.nextLine());
                }
                return result;
        }
}

注釈

If you use try-with-resources, all the resources will be correctly closed, even if exceptions are thrown, including during initialization.

try-with-resourcesを使えば、初期化時を含めて途中で例外が発生しても正しくクローズしてくれます。

New Style¶

If you use java.nio.file.Files class, you are able to write your programs simply, as following.

java.nio.file.Files クラスを使うと、次のようにシンプルにプログラムを作ることが出来ます

/** New style (Java 7) **/
public static List<String> readFromFile(String fileName) throws IOException {
        return Files.readAllLines(Paths.get(fileName));
}

注釈

  • Files: java.nio.file.Files
  • Paths: java.nio.file.Paths

参考

Java8

In Java 8, it can process to read line-by-line, using Stream API.

Java 8ではStream APIを利用して1行ずつ読み込む処理を行うことが出来ます。

Handling Objects More Easily / オブジェクト操作をより容易に¶

java.util.Objects provides static utility methods for operating on objects including null-safe or null-tolerance methods.

Comparing Two Objects¶

Objects.equals(a, b) is the replacement of a.equals(b). Objects.equals() is null tolerant.

a b Objects.equals(a, b)
non-null non-null a.equals(b)
non-null null false
null non-null false
null null true

Old Style¶

if ("enabled".equals(param)) { // param can be null
        // do something
}

or

if (param != null && param.equals("enabled") {
        // do something
}

New Style¶

if (Objects.equals(param, "enabled")) {
        // do something
}

Generating Hash Codes¶

Objects.hash() is the utility method to generate hash code.

Old Style¶

public class Dto {
        private final long id;
        private final String hoge;
        private final String fuga;

        @Override
        public int hashCode() { // generated by IDE
                final int prime = 31;
                int result = 1;
                result = prime * result + ((hoge == null) ? 0 : fuga.hashCode());
                result = prime * result + (int) (id ^ (id >>> 32));
                result = prime * result + ((name == null) ? 0 : hoge.hashCode());
                return result;
        }
}

New Style¶

public class Dto {
        private final long id;
        private final String hoge;
        private final String fuga;

        @Override
        public int hashCode() {
                return Objects.hash(id, hoge, fuga);
        }
}

Converting to String¶

Use Objects.toString(o, nullDefault).

Old Style¶

Dto dto = dao.findById(100);
System.out.println(dto == null ? "[No record found]" : dto.toString());

New Style¶

Dto dto = dao.findById(100);
System.out.println(Objects.toString(dao, "[No record found]"));

Constructor with Non-Null-Required Parameters¶

Objects.requireNonNull() will provide you to check if the parameters is null.

Old Style (Too complex code)¶

public class Name {
        private final String firstName;
        private final String familyName;

        public Name(String firstName, String familyName) {
                if (firstName == null) {
                        throw new NullPointerException("firstName must not be null");
                }
                if (familyName == null) {
                        throw new NullPointerException("familyName must not be null");
                }
                this.firstName = firstName;
                this.familyName = familyName;
        }

        @Override
        public String toString() {
                return firstName.toString() + " " + familyName.toString(); // safe from NullPointerException
        }
}

Old Style (Unsafe code)¶

public class Name {
        private final String firstName;
        private final String familyName;

        public Name(String firstName, String familyName) {
                this.firstName = firstName;
                this.familyName = familyName;
        }

        @Override
        public String toString() {
                return firstName.toString() + " " + familyName.toString(); // NullPointerException can be raised
        }
}

New Style¶

public class Name {
        private final String firstName;
        private final String familyName;

        public Name(String firstName, String familyName) {
                this.firstName = Objects.requireNonNull(firstName, "firstName must not be null");
                this.familyName = Objects.requireNonNull(familyName, "familyName must not be null");
        }

        @Override
        public String toString() {
                return firstName.toString() + " " + familyName.toString(); // safe from NullPointerException
        }
}

Handling Collections / コレクション操作¶

Making New Lists / リストを作成する¶

Old Style¶

List<Integer> numbers = new ArrayList<Integer>();
numbers.add(1);
numbers.add(3);
numbers.add(5);
numbers.add(7);
numbers.add(9);

New Style¶

List<Integer> numbers = Arrays.asList(1, 3, 5, 7, 9);

You can make lists of any types, as well as Integer.

Integer型の他、任意の型のリストを作ることが出来ます。

List<Hoge> hogeList = Arrays.asList(
        new Hoge(...),
        new Hoge(...),
        new Hoge(...),
        ...
);

注釈

Arrays.asList() will generate lists that do not allow addition. It is impossible to call .add() and/or .remove(); UnsupportedOperationException exception occurs.

Arrays.asList()で生成したリストは追加不能なリストです。 .add()や.remove()などを呼び出すことは出来ません（UnsupportedOperationException例外が発生します）。

Immutable Collections / 変更不能なコレクション¶

If you want to block the modification of the list defensively, set, and map, you can use those collections that do not allow modification after once created.

New Style¶

List<Integer> mutableList = new ArrayList<Integer>();
mutableList.add(1);
mutableList.add(3);
mutableList.add(5);

// make immutable list from mutable list
List<Integer> immutableList = Collections.unmodifiableList(mutableList);

immutableList.add(7); // will result in UnsupportedOperationException

警告

オリジナルの変更可能リストに対する変更は、生成された変更不能リストに対しても反映されてしまいます。

The changes for original mutable lists will also affect on the generated immutable lists.

List<Integer> mutableList = new ArrayList<Integer>();
mutableList.add(1);
mutableList.add(3);
mutableList.add(5);

List<Integer> immutableList = Collections.unmodifiableList(mutableList);
System.out.println(immutableList); // [1, 3, 5]

mutableList.add(7);
System.out.println(immutableList); // [1, 3, 5, 7]

メソッド引数でリストが渡された場合などは特にですが、必要に応じてリストのコピーを作成する必要があります。

You have to create new copies of the lists, if necessary, especially when mutable lists are given via method arguments.

List<Integer> mutableList = new ArrayList<Integer>();
mutableList.add(1);
mutableList.add(3);
mutableList.add(5);

List<Integer> immutableList
                = Collections.unmodifiableList(new ArrayList(mutableList));
System.out.println(immutableList); // [1, 3, 5]

mutableList.add(7);
System.out.println(immutableList); // still [1, 3, 5]

Typical Usage¶

典型的な利用方法としては、作業用の変更可能リストからメソッドの戻り値として変更不能リストを生成することが挙げられます。

One of the typical usages of the immutable collections is to generate immutable lists as returned values from the methods, from mutable work lists.

public List<String> fizzBuzz(int to) {
        List<String> result = new ArrayList<String>();
        for (int i = 1; i < to; i++) {
                if (i % 15 == 0) {
                        result.add("FizzBuzz");
                } else if (i % 3 == 0) {
                        result.add("Fizz");
                } else if (i % 5 == 0) {
                        result.add("Buzz");
                } else {
                        result.add(Integer.toString(i));
                }
        }
        // Finally, make a result list as an immutable list.
        return Collections.unmodifiableList(result);
}

Method List¶

You can make unmodifiable views of (general) collections, maps, sets, sorted maps, and sorted sets, as well as that of lists.

  • Collections.unmodifiableCollection()
  • Collections.unmodifiableList()
  • Collections.unmodifiableMap()
  • Collections.unmodifiableSet()
  • Collections.unmodifiableSortedMap()
  • Collections.unmodifiableSortedSet()

Empty Collections / 空のコレクション¶

You can get empty collections:

Old Style¶

List<String> emptyList = new ArrayList<String>();
Set<Integer> emptySet = new HashSet<Integer>();

New Style¶

List<String> emptyList = Collections.emptyList();
Set<Integer> emptySet = Collections.emptySet();

注釈

As Collections.emptyXXX() returns an immutable collection, you are not allowed to execute any mutable operations.

Method List¶

  • Collections.emptyList()
  • Collections.emptySet()
  • Collections.emptyMap()

EnumSet¶

列挙型のセットにはEnumSetを利用します。内部的にビットフラグが利用されており、HashSetよりも高速に動作します。また、EnumSetはセットを生成するための有用なメソッドを提供しています。

Use EnumSet to generate enum-type sets. It works faster than HashSet, and provides several useful methods.

enum Weekday { SUNDAY, MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY }

Old Style¶

Set<Weekday> weekdays = new HashSet<Weekday>();
weekdays.add(Weekday.MONDAY);
weekdays.add(Weekday.TUESDAY);
weekdays.add(Weekday.WEDNESDAY);
weekdays.add(Weekday.THURSDAY);
weekdays.add(Weekday.FRIDAY);

Set<Weekday> holidays = new HashSet<Weekday>();
holidays.add(Weekday.SUNDAY);
holidays.add(Weekday.SATURDAY);

New Style¶

Set<Weekday> weekdays = EnumSet.of(
                Weekday.MONDAY,
                Weekday.TUESDAY,
                Weekday.WEDNESDAY,
                Weekday.THURSDAY,
                Weekday.FRIDAY
);

Set<Weekday> holidays = EnumSet.of(
                Weekday.SUNDAY,
                Weekday.SATURDAY
);

注釈

EnumSetはSetのサブクラスなので、weekdays、holidaysはEnumSet<Weekday>として宣言しても構いません。

It is okay to declare “weekdays” and “holidays” as EnumSet<Weekday>, for EnumSet is a subclass of Set.

「○○から××まで」というセットを作成する場合にはEnumSet.range()を使うことが出来ます。また、「△△以外」というセットを作成する場合には、EnumSet.complementOf()を使うことが出来ます。

次のソースコードは同じセットを生成します。

You can use EnumSet.range() to create sets like “from XX to YY.” Also, you can use EnumSet.complementOf() to create sets like “except ZZZ.”

Following codes generate the same set.

// weekdays are from monday to friday
EnumSet<Weekday> weekdays = EnumSet.range(Weekday.MONDAY, Weekday.FRIDAY);

// holidays are non-weekday
Set<Weekday> holidays = EnumSet.complementOf(weekdays);

注釈

EnumSet.complementOf()は、EnumSetを引数に取ります。そのため、weekdaysはSet<Weekday>ではなく、EnumSet<Weekday>として宣言しています。

EnumSet.complementOf() requires an argument of EnumSet. That’s why weekdays is declared as EnumSet<Weekday>, not Set<Weekday>.

Method List¶

  • EnumSet.of(e1, e2, ...)
  • EnumSet.noneOf()
  • EnumSet.allOf()
  • EnumSet.range()
  • EnumSet.copyOf(): useful to convert from a normal Set (Collection) to a EnumSet.
  • EnumSet.complementOf()

EnumMap¶

列挙型をキーとしたマップに最適化されたEnumMapクラスがあります。使い方は他のMap (HashMap、TreeMapなど)と同じです。

EnumMap is a specialized Map implementation for use with enum type keys. Usage of EnumMap is the same as other map implementations such as HashMap and TreeMap.

It suppress warning. But throw ClassCastException.

@SuppressWarnings("unchecked")
Hoge hoge = (Hoge) arg;

New Style¶

Generics / ジェネリクス¶

Declaring Generic Methods / ジェネリックなメソッドを宣言する¶

If you want to make a versatile method about types, define generic method with type parameters.

public static <T> void addToList(List<T> list, T element) {
        list.add(element);
}

In the above example, T is the type parameter, which is declared with surrounded <>. Type parameter T will be bound to any types.

public static void main(String[] args) {
        List<String> strings = new ArrayList<>();
        addToList(strings, "abc"); // type parameter T is bound to String

        List<Integer> numbers = new ArrayList<>();
        addToList(numbers, 1); // type parameter T in bound to Integer
}

Declaring Generic Classes / ジェネリックなクラスを宣言する¶

You can define the generic class with type parameters.

public class Pair<A, B> { // two type parameter A and B are defined.
        private final A a;
        private final B b;

        public Pair(A a, B b) {
                this.a = a;
                this.b = b;
        }

        public A getA() { // you don't need to define type parameter like `<A>`, it is already defined above.
                return a;
        }

        public B getB() {
                return b;
        }

        @Override
        public String toString() {
                return "(" + a + ", " + b + ")";
        }

        @Override
        public boolean equals(Object o) {
                (OMIT)
        }

        @Override
        public int hashCode() {
                (OMIT)
        }
}

In the class Pair, two type parameter A and B are defined. A and B are independently bound to concrete types.

public static void main(String[] args) {
        Pair<String, Integer> stringIntegerPair = new Pair<>("abc", 10); // A is bound to String, B is bound to Integer
        System.out.println(stringIntegerPair);

        Pair<Integer, Integer> integerPair = new Pair<>(10, 20); // both A and B are bound to Integer
        System.out.println(integerPair);
}

Use unbounded wildcard(?)¶

You should limit type with type parameters, if you can. But If you can’t do this. You can use wildcard(?).

This topic will help you when you need to conversion under java1.4 codes.

Old Style¶

private List list = new ArrayList();

public Object exists(Object target) {
  if (!(target instanceof Hoge)) {
    return null;
  }

  for (int ii = 0; ii < list.size(); ii++) {
    Object temp = list.get(ii);
    if (temp instanceof Hoge) {
      if (((Hoge) temp).same(target)) {
        return temp;
      }
    }
  }

  return null;
}

New Style(Better)¶

private List<Hoge> list = new ArrayList<>();

public Hoge exists(Hoge target) {
  for (Hoge temp : list) {
    if (temp.same(target)) {
      return temp;
    }
  }

  return null;
}

New Style¶

If you can’t beleve argument type is unique(Hoge).

private List<?> list = new ArrayList<>();

public Hoge exists(Object target) {
  if (!(target instanceof Hoge)) {
    return null;
  }

  for (Object temp : list) {
    if (temp instanceof Hoge) {
      if (Hoge.class.cast(temp).same(target)) {
        return temp;
      }
    }
  }

  return null;
}

Actually, in this case it is OK to use Object.

private List<Object> list = new ArrayList<>();

But if you need to add @SuppressWarnings, you should use wildcard.

Use @SuppressWarnings¶

public void someMethod(Object value) {
  if (value instanceof List) {
    @SuppressWarnings("unchecked")
    List<Object> list = List.class.cast(value);
      :
      :
      :
  }
}

Use wildcard¶

public void someMethod(Object value) {
  if (value instanceof List) {
    List<?> list = List.class.cast(value);
      :
      :
      :
  }
}
